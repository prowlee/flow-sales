# FlowSales 本番稼働ロードマップ

このドキュメントは、AI SDRシステム「FlowSales」を本番稼働させ、実際に商談を安定して獲得するために必要なステップを管理します。

## Phase 1: 堅牢な自動化 (基盤強化) - **完了**
- [x] **メインループ（司令塔）の実装**: `src/index.ts` で定期的に（12時間おき）実行するスケジュール機能を実装済み。
- [x] **エラーハンドリングの強化**: 各リードごとのエラーをキャッチし、DBの `errorLog` に記録して続行する仕組みを実装済み。
- [x] **高度なロギング**: コンソールおよびダッシュボードでのステータス可視化を実装。

## Phase 2: セールス品質の向上 (成果の最大化) - **完了**
- [x] **リサーチプロンプトの洗練**: 技術選定の背景やビジネスの本質を深掘りするようプロンプトを大幅強化済み。
- [x] **パーソナライズの最適化**: AIっぽさを排除し、エンジニア同士の対話トーンを実現するプロンプト定数化済み。
- [x] **マルチテンプレート/スタイル対応**: TECHNICAL/BUSINESSの2スタイルを自動選択する機能を `PersonalizationService` に実装済み。

## Phase 3: 運用の効率化 (使いやすさ) - **完了**
- [x] **承認用インターフェース**: `/dashboard` にて未送信メールの確認・承認・却下（再試行）ができる画面を実装済み。
- [x] **Slack通知連携**: 承認待ちの発生やエラーを通知する `SlackService` を実装済み。
- [x] **レポート機能**: ダッシュボードに全体の統計（送信数、失敗数、Gross Leads等）を表示するレポート機能を実装済み。

## Phase 4: スケールとコンプライアンス - **完了**
- [x] **ドメイン保護**: `DAILY_SEND_LIMIT` による流量制限と、送信間のランダムウェイトによるセーフティ・スロットルを実装済み。
- [x] **複数アカウント対応**: InstantlyのキャンペーンIDベースで動作するため、Instantly側でのアカウントローテーションに対応。
- [x] **オプトアウト管理**: `unsubscribed` フラグをDBに追加し、配信停止希望者への送信を自動でスキップする仕組みを実装済み。
