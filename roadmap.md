# FlowSales 本番稼働ロードマップ

このドキュメントは、AI SDRシステム「FlowSales」を本番稼働させ、実際に商談を安定して獲得するために必要なステップを管理します。

## Phase 1: 堅牢な自動化 (基盤強化)
- [ ] **メインループ（司令塔）の実装**: `src/index.ts` で定期的にリードを取得・処理し続けるスケジュール機能を実装する。
- [ ] **エラーハンドリングの強化**: 特定のリードでエラー（サイトが落ちている、API制限など）が出てもプロセスを止めず、失敗理由をDBに記録して次へ進むように改良する。
- [ ] **高度なロギング**: どのステップで何が起きているか（リサーチ結果、作成されたメール等）を後から追いやすいよう、ログ出力を整理する。

## Phase 2: セールス品質の向上 (成果の最大化)
- [ ] **リサーチプロンプトの洗練**: 表面的な技術情報だけでなく、その企業のビジネスモデルや解決しようとしている課題をより深く洞察するためのプロンプト調整。
- [ ] **パーソナライズの最適化**: AI特有の言い回しを排除し、ターゲット（CTO/Founder）が「自分のために書かれた」と感じるトーンの作り込み。
- [ ] **マルチテンプレート対応**: ターゲットの属性に合わせて、訴求軸を変えたメールを生成できるようにする。

## Phase 3: 運用の効率化 (使いやすさ)
- [ ] **承認用インターフェース**: DBを直接操作せずに、未送信メールの確認・修正・一括承認ができる仕組み（簡易Web画面やスプレッドシート連携）。
- [ ] **Slack通知連携**: 新しいリードの追加、承認待ちメールの発生、商談獲得やエラーなどの重要イベントをSlackへ通知。
- [ ] **レポート機能**: 送信数、開封率、返信率を可視化し、改善サイクルを回せるようにする。

## Phase 4: スケールとコンプライアンス
- [ ] **ドメイン保護**: 送信元メールアドレスの評判（Reputation）を守るための、送信頻度やウォームアップの自動制御。
- [ ] **複数アカウント対応**: 送信ボリュームを増やすために、複数のInstantlyアカウントやキャンペーンを統合管理できるようにする。
- [ ] **オプトアウト管理**: 配信停止希望者の自動除外と、ブラックリスト管理の堅牢化。
